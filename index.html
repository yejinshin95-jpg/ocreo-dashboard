9월까지 꼭 이어져야 하는 순서

여유 시간 확보

토스 결제 완성 → 증빙서류 자동 발행 → 9월 신청 시작

증빙서류(세금계산서 등) 자동 발행은 토스 결제가 먼저 완성되어야 만들 수 있습니다. 토스 작업을 7월에 먼저 시작하기로 결정(7/4)해서 8월 중순 완성 → 증빙서류 기능을 만들 시간 약 4주를 확보했습니다. 토스 수수료는 1.8%로 낮추기로 협의됨(6/9). 토스는 세금계산서 발행이 불가함이 공식 확인되어(6/26 메일) 발행처는 바로빌로 확정.

관리 중 · 문제 없음

8~9월에 일이 몰립니다: 23회차 신청서 + 토스 마무리 + 타이페이 준비

페어 관리 기능이 새로 추가되어 7월 개발 부하도 늘었습니다 — 그래서 정산 관리는 지금 화면에서 가능한 범위(1단계)만 먼저 진행합니다. 8월에는 급하지 않은 사이트 수정 요청을 잠시 미뤄 개발팀이 큰 작업에 집중하도록 하고, 환불·환급 관리 기능은 9월 초 완성으로 여유를 둡니다. 타이페이 준비(9월)가 새로 추가되어, 8월 말에 담당 배분을 한 번 더 점검하겠습니다.

💬 기획자 조언 — 위 목록 외에 기획·정책 결정이 필요한 것들

① 기업 정보 인증 정책 (7월 중) — 기업 정보를 별도로 만드는 구조 개선(OCH-1452)이 7월에 개발되는데, "기업 인증 단계를 둘지"가 아직 미정(6/30 논의). 개발 완료 전에 정책을 정해야 재작업이 없습니다.

② 증빙 신청 운영 세부 정책 (9월) — 작가 시작·마감일의 실제 날짜, 마감 임박 알림 여부, 마감 후 뒤늦은 요청 처리 기준. 기능 개발과 별개로 사무국 운영 규칙 문서가 필요합니다.
